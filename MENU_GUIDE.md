## 📊 資料管理

### 郵件資料結構
```
entry_id: 郵件唯一識別碼
subject: 郵件主旨
sender_name: 寄件者姓名
sender_email: 寄件者信箱
received_time: 接收時間
body: 郵件內容（已限制長度）
folder: 資料夾名稱
```

### 執行記錄
每次執行的統計資料會儲存在 `execution_log.json`：
```json
{
  "timestamp": "2026-02-12T14:30:00",
  "plan": "⚡ 日常使用",
  "emails_processed": 50,
  "emails_added": 50,
  "emails_skipped": 0,
  "elapsed_time": 1.8,
  "avg_time_per_email": 0.036
}
```

## ⚙️ 設定說明

### 預設設定
```python
DB_NAME = 'emails.db'              # 資料庫檔案名稱
FOLDER_NAME = '收件匣'              # Outlook 資料夾名稱
ACCOUNT_NAME = 'your@email.com'    # 您的郵件帳號
```

### 自訂設定選項
- **郵件數量**: 1-1000 封
- **內文長度**: 50-2000 字元
- **處理優先順序**: 最新郵件優先

## 🔍 故障排除

### 常見問題

#### Q: 連接 Outlook 失敗
**A:**
1. 確認 Outlook 已開啟
2. 確認郵件帳號正確
3. 重新啟動 Outlook

#### Q: 處理速度很慢
**A:**
1. 選擇更快的方案
2. 減少處理郵件數量
3. 關閉其他程式

#### Q: 郵件重複
**A:**
1. 工具會自動跳過已存在的郵件
2. 這是正常現象，表示資料庫已有這些郵件

#### Q: 中文顯示亂碼
**A:**
1. 工具已內建編碼修復
2. 如果仍有問題，重新執行測試
